မနေ့ကတော့ Facebook မှာ Post တွေတင်တာ ဖျတ်တာ နဲ့ ပက်သက်ပြီး HTTP Verb တွေ ဖြစ်တဲ့ GET,POST,PUT,DELETE အကြောင်း ဆွေးနွေးခဲ့ပြီးပါပြီ။

ဒီနေ့မှာတော့ နောက်ဆုံးအပိုင်းအနေနဲ့ Body / Payload တွေ အကြောင်း ဆက်လက်ဆွေးနွေးသွားပါမယ်။

အရင်က ပြောခဲ့တယ်။
HTTP Request တစ်ခုမှာ

1. URL
2. HTTP Verb 

ပါရမယ်လို့။ 

ဒီနေ့တော့ တစ်ခု ထပ်တိုးကြရအောင်။

1. URL
2. HTTP Verb
3. Payload

Payload ဆိုတာကတော့ ရထားနဲ့ ဉပမာ ပေးမယ်ဆိုရင် ရထားတွဲလို့ ပြောရမယ်။

Facebook မှာ Post တင်တဲ့ အကြောင်းကို ပြန်တွေးကြည့်ရအောင်

POST api/v1/posts 
ဒီမှာ ဆိုရင် (၂) ပိုင်းပဲ ရှိသေးတယ်။

URL api/v1/posts ကို
HTTP Post လုပ်မယ်။

ရထားတစ်စင်း Facebook Server ကို လွှတ်လိုက်မယ်ပေါ့။
အဲ့ရထားမှာ ဘာလိုသေးလဲ?

ကျွန်တော်တို့ တင်ချင်တဲ့ စာတွေ ပုံတွေ ဗီဒီယိုတွေ ဒါတွေ ထည့်ပေးလိုက်ဖို့ လိုသေးတယ်။
ဒါကို payload လို့ ခေါ်တယ်။

ဒီတော့ "မင်္ဂလာပါ ချစ်တို့ရေ" ဆိုပြီး Post တင်ချင်တယ်ဆိုကြပါစို့။
ဒီတော့ HTTP Request က ဒီလို ဖြစ်သွားမယ်။

URL api/v1/posts
HTTP Verb : POST
Payload :  {"message":"မင်္ဂလာပါ ချစ်တို့ရေ"}


ကိုယ့်ဘက်က Payload တွေ တင်ပေးလိုက်သလိုပဲ Facebook ဘက်ကလည်း တစ်ဖန်ပြန်ပြီး ထည့်ပေးလိုက်မယ့် အချက်အလက်တွေရှိတယ်။ ဒါကိုတော့ ပြန်လာတဲ့ ရထား Response ရဲ့ ရထားတွဲတွေပေါ်မှာ ကြည့်လို့ရမယ်။ စာမှာတော့ Resposne Body လို့ ခေါ်တယ်။

HTTP Response က ဒါမျိုး ဖြစ်မယ်။
Status Code : 200
Body : { "message" : "post တင်ပြီးပါပြီ" }

Response နဲ့ ပက်သက်လို့က (၂) ပိုင်းပဲ မှတ်ထားရုံပဲ။

Status Code နဲ့ Body

Status Code ဆိုတာကတော့ ကိုယ် Request လုပ်လိုက်တဲ့ Resource / Operation အောင်မြင် / မအောင်မြင် ဆိုတာကို ပြပေးတဲ့ နံပါတ်။

200 - အိုကေတယ်လို့ ပြောတာ။
404 - ကိုယ်လိုချင်တဲ့ Resource သူတို့စီမှာ မရှိဘူး။
403 - ကိုယ်ကြည့်ခွင့်မရှိတဲ့ Resource
500 - ဆာဗာကိုယ်တိုင် ရွာလည်နေတယ်။ 

အကြမ်းဖျဉ်း အားဖြင့် ဒီလောက်ဆိုရပြီ။

ဒီနေ့အတွက် မှတ်ထားဖို့ လိုတာက 

Request လုပ်တဲ့အခါ ကိုယ့်စီက ဒေတာတွေ ထည့်ပေးလိုက်ချင်ရင် Payload လို့ ခေါ်တဲ့ နေရာကနေ ထည့်ပေးလိုက်လို့ရတယ်။
Response ကို ကြည့်တဲ့အခါ Status Code နဲ့ Body ကို ကြည့်ရမယ်။

ဒီလောက်ဆိုရင် ကုတ်တစ်ကြောင်းမှ မရေးခင်မှာ Restful Web Service ဆိုတာ ဘယ်လို အလုပ်လုပ်နေကြလဲ မျက်လုံးထဲမှာ မြင်မယ်လို့ ယုံကြည်ပါတယ်။ နောက်တစ်ပိုင်းမှာတော့ မျက်လုံးထဲမှာ မြင်နိုင်မယ့် တကယ် Restful Web Service တစ်ခုကို စမ်းသပ် ကြည့်ကြပါမယ်။
